# Transformer 模型参数调节指南

---

## 一、推理/生成参数（`model.generate()`）

控制模型生成文本的行为，无需重新训练。

| 参数 | 说明 | 默认值 | 调节建议 |
|------|------|--------|---------|
| `max_new_tokens` | 最大生成 token 数 | 20 | 根据任务需要设置，长文本可设 512~2048 |
| `min_new_tokens` | 最小生成 token 数 | 0 | 避免过短输出时设置 |
| `temperature` | 随机性控制，越低越保守 | 1.0 | 创意任务用 0.7~1.0，精确任务用 0.1~0.3 |
| `top_p` | nucleus sampling，累积概率阈值 | 1.0 | 通常设 0.85~0.95，与 temperature 配合使用 |
| `top_k` | 每步只从概率最高的 k 个 token 中采样 | 50 | 设 0 关闭，设 50 是常用值 |
| `do_sample` | 是否启用随机采样 | False | True=采样，False=贪婪解码 |
| `num_beams` | beam search 的束宽 | 1 | 设 4~8 可提升质量，但速度变慢 |
| `repetition_penalty` | 重复惩罚，>1 降低重复 | 1.0 | 1.1~1.3 可减少重复内容 |
| `length_penalty` | 对输出长度的惩罚（beam search 时有效） | 1.0 | >1 鼓励长输出，<1 鼓励短输出 |
| `no_repeat_ngram_size` | 禁止重复的 n-gram 大小 | 0 | 设 3 可避免 3 个词的重复片段 |
| `early_stopping` | beam search 时遇到 EOS 是否提前停止 | False | beam search 时建议设 True |

```python
from transformers import AutoModelForCausalLM, AutoTokenizer

tokenizer = AutoTokenizer.from_pretrained("gpt2")
model = AutoModelForCausalLM.from_pretrained("gpt2")

inputs = tokenizer("Once upon a time", return_tensors="pt")

output = model.generate(
    **inputs,
    max_new_tokens=200,
    temperature=0.8,
    top_p=0.92,
    top_k=50,
    do_sample=True,
    repetition_penalty=1.2,
    no_repeat_ngram_size=3,
)
print(tokenizer.decode(output[0], skip_special_tokens=True))
```

## 二、训练参数（`TrainingArguments`）

控制模型微调过程，需要训练数据。

| 参数 | 说明 | 默认值 | 调节建议 |
|------|------|--------|---------|
| `learning_rate` | 学习率 | 5e-5 | 微调通常用 1e-5 ~ 5e-5 |
| `num_train_epochs` | 训练轮数 | 3 | 小数据集可设 5~10，大数据集 1~3 |
| `per_device_train_batch_size` | 每个 GPU 的 batch 大小 | 8 | 显存不足时减小，配合梯度累积 |
| `gradient_accumulation_steps` | 梯度累积步数 | 1 | 模拟更大 batch，显存不足时设 4~8 |
| `warmup_steps` | 学习率预热步数 | 0 | 通常设总步数的 5%~10% |
| `weight_decay` | L2 正则化系数 | 0 | 0.01~0.1 可防止过拟合 |
| `lr_scheduler_type` | 学习率调度策略 | linear | cosine 通常效果更好 |
| `save_steps` | 每隔多少步保存一次 | 500 | 根据训练时长调整 |
| `eval_steps` | 每隔多少步评估一次 | 500 | 与 save_steps 保持一致 |
| `fp16` / `bf16` | 混合精度训练 | False | GPU 支持时设 True，节省显存约 50% |
| `max_grad_norm` | 梯度裁剪阈值 | 1.0 | 训练不稳定时降低到 0.5 |

```python
from transformers import Trainer, TrainingArguments

training_args = TrainingArguments(
    output_dir="./output",
    num_train_epochs=3,
    per_device_train_batch_size=4,
    gradient_accumulation_steps=4,   # 等效 batch_size=16
    learning_rate=2e-5,
    warmup_steps=100,
    weight_decay=0.01,
    lr_scheduler_type="cosine",
    fp16=True,
    save_steps=500,
    eval_steps=500,
    logging_steps=50,
)

trainer = Trainer(
    model=model,
    args=training_args,
    train_dataset=train_dataset,
    eval_dataset=eval_dataset,
)
trainer.train()
```

## 三、LoRA 参数（PEFT 高效微调）

只训练少量附加参数，显存占用大幅减少。

| 参数 | 说明 | 默认值 | 调节建议 |
|------|------|--------|---------|
| `r` | LoRA 秩，控制可训练参数量 | 8 | 越大效果越强但越慢，通常 8~64 |
| `lora_alpha` | 缩放因子，通常设为 r 的 2 倍 | 16 | 设为 2*r，如 r=16 则 alpha=32 |
| `lora_dropout` | LoRA 层的 dropout | 0.0 | 小数据集设 0.05~0.1 防过拟合 |
| `target_modules` | 作用于哪些模块 | 模型相关 | 常用 `q_proj`, `v_proj`, `k_proj`, `o_proj` |
| `bias` | 是否训练 bias | none | 通常保持 none |

```python
from peft import LoraConfig, get_peft_model, TaskType

lora_config = LoraConfig(
    task_type=TaskType.CAUSAL_LM,
    r=16,
    lora_alpha=32,
    lora_dropout=0.05,
    target_modules=["q_proj", "v_proj", "k_proj", "o_proj"],
    bias="none",
)

model = get_peft_model(model, lora_config)
model.print_trainable_parameters()
# 输出示例：trainable params: 4,194,304 || all params: 6,742,609,920 || trainable%: 0.06
```

## 四、量化参数（节省显存）

加载时压缩模型精度，无需训练。

| 参数 | 说明 | 调节建议 |
|------|------|---------|
| `load_in_8bit` | INT8 量化 | 显存减少约 50%，精度损失极小 |
| `load_in_4bit` | INT4 量化 | 显存减少约 75%，精度有一定损失 |
| `bnb_4bit_compute_dtype` | 计算时的数据类型 | 设 `torch.float16` 或 `bfloat16` |
| `bnb_4bit_use_double_quant` | 双重量化，进一步节省显存 | 通常设 True |
| `bnb_4bit_quant_type` | 量化类型 | `nf4` 通常优于 `fp4` |

```python
import torch
from transformers import AutoModelForCausalLM, BitsAndBytesConfig

# 4bit 量化
bnb_config = BitsAndBytesConfig(
    load_in_4bit=True,
    bnb_4bit_compute_dtype=torch.float16,
    bnb_4bit_use_double_quant=True,
    bnb_4bit_quant_type="nf4",
)

model = AutoModelForCausalLM.from_pretrained(
    "meta-llama/Llama-2-7b-hf",
    quantization_config=bnb_config,
    device_map="auto",
)
```

## 五、模型结构参数（`AutoConfig`）

从头训练或修改模型架构时使用。

| 参数 | 说明 | 调节建议 |
|------|------|---------|
| `hidden_size` | 隐藏层维度 | 越大越强，但计算量和显存成倍增加 |
| `num_hidden_layers` | Transformer 层数 | 减少层数可加速推理 |
| `num_attention_heads` | 注意力头数量 | 需整除 hidden_size |
| `intermediate_size` | FFN 中间层大小 | 通常为 hidden_size 的 4 倍 |
| `hidden_dropout_prob` | 隐藏层 dropout | 0.1~0.2，数据少时适当增大 |
| `attention_probs_dropout_prob` | 注意力 dropout | 同上 |
| `max_position_embeddings` | 最大序列长度 | 根据任务的文本长度设置 |

```python
from transformers import AutoConfig, AutoModelForCausalLM

config = AutoConfig.from_pretrained("gpt2")
config.n_layer = 6                    # 减少层数
config.hidden_dropout_prob = 0.1      # 增加正则化
config.max_position_embeddings = 2048 # 扩展上下文长度

model = AutoModelForCausalLM.from_config(config)
```

## 六、Qwen 模型参数调节

Qwen 系列（Qwen2.5 等）基于 Hugging Face Transformers，但有自己的官方推荐参数和使用方式。

### Qwen 官方默认生成参数（来自 generation_config.json）

| 参数 | 官方默认值 | 说明 |
|------|-----------|------|
| `temperature` | **0.7** | 比通用默认值(1.0)更保守 |
| `top_p` | **0.8** | 较严格的 nucleus sampling |
| `top_k` | **20** | 较小的候选集，输出更稳定 |
| `repetition_penalty` | **1.05** | 轻微抑制重复 |
| `do_sample` | **True** | 默认启用采样 |

### 基础推理代码

```python
from transformers import AutoModelForCausalLM, AutoTokenizer

model_name = "Qwen/Qwen2.5-7B-Instruct"

model = AutoModelForCausalLM.from_pretrained(
    model_name,
    torch_dtype="auto",   # 自动选择 float16/bfloat16
    device_map="auto",    # 自动分配 GPU/CPU
)
tokenizer = AutoTokenizer.from_pretrained(model_name)

# Qwen 必须使用 chat_template 格式
messages = [
    {"role": "system", "content": "You are a helpful assistant."},
    {"role": "user", "content": "请介绍一下大语言模型。"}
]

text = tokenizer.apply_chat_template(
    messages,
    tokenize=False,
    add_generation_prompt=True
)

inputs = tokenizer([text], return_tensors="pt").to(model.device)

output = model.generate(
    **inputs,
    max_new_tokens=512,
    temperature=0.7,        # Qwen 官方推荐值
    top_p=0.8,              # Qwen 官方推荐值
    top_k=20,               # Qwen 官方推荐值
    repetition_penalty=1.05,
    do_sample=True,
)

# 只解码新生成的部分
generated_ids = output[0][len(inputs.input_ids[0]):]
print(tokenizer.decode(generated_ids, skip_special_tokens=True))
```

### Qwen 场景化参数推荐

| 场景 | temperature | top_p | top_k | 说明 |
|------|------------|-------|-------|------|
| 日常对话 | 0.7 | 0.8 | 20 | 官方默认，平衡流畅与多样 |
| 代码生成 | 0.2 | 0.9 | 40 | 低温度保证准确性 |
| 创意写作 | 0.9 | 0.95 | 50 | 高温度增加创意 |
| 翻译/摘要 | 0.1 | 0.9 | 0 | 接近贪婪，输出稳定 |
| 数学推理 | 0.0 | 1.0 | 1 | 完全贪婪，确定性输出 |

### Qwen + LoRA 微调（显存不足时推荐）

```python
import torch
from transformers import AutoModelForCausalLM, AutoTokenizer, BitsAndBytesConfig
from peft import LoraConfig, get_peft_model, TaskType

# 4bit 量化加载，节省显存
bnb_config = BitsAndBytesConfig(
    load_in_4bit=True,
    bnb_4bit_compute_dtype=torch.bfloat16,  # Qwen 推荐 bfloat16
    bnb_4bit_use_double_quant=True,
    bnb_4bit_quant_type="nf4",
)

model = AutoModelForCausalLM.from_pretrained(
    "Qwen/Qwen2.5-7B-Instruct",
    quantization_config=bnb_config,
    device_map="auto",
)

# Qwen2.5 的注意力模块名称
lora_config = LoraConfig(
    task_type=TaskType.CAUSAL_LM,
    r=16,
    lora_alpha=32,
    lora_dropout=0.05,
    target_modules=[
        "q_proj", "k_proj", "v_proj", "o_proj",  # 注意力层
        "gate_proj", "up_proj", "down_proj",       # FFN 层
    ],
    bias="none",
)

model = get_peft_model(model, lora_config)
model.print_trainable_parameters()
```

### Qwen 长文本支持（上下文超过 32K）

Qwen2.5 最大支持 **128K tokens**，超过 32K 需在 `config.json` 中开启 YaRN：

```python
# 加载时动态开启长文本支持
model = AutoModelForCausalLM.from_pretrained(
    "Qwen/Qwen2.5-7B-Instruct",
    torch_dtype="auto",
    device_map="auto",
    # 覆盖 rope_scaling 配置
    rope_scaling={"type": "yarn", "factor": 4.0, "original_max_position_embeddings": 32768}
)
```

## 七、在已训练模型上微调的关键参数

在预训练模型基础上微调（Fine-tuning），核心原则是**"轻动"**——用小学习率、少轮次，在不破坏原有知识的前提下适配新任务。

### 最重要的参数（按优先级排列）

#### 1. 学习率（最关键）

微调学习率必须远小于预训练，否则会"灾难性遗忘"原有能力。

| 微调方式 | 推荐学习率 |
|---------|-----------|
| 全量微调（Full Fine-tuning） | `1e-5 ~ 5e-5` |
| LoRA 微调 | `1e-4 ~ 3e-4` |
| 只训练分类头 | `1e-3 ~ 1e-2` |

```python
# 不同模块设置不同学习率（分层学习率）
optimizer = torch.optim.AdamW([
    {"params": model.base_model.parameters(), "lr": 1e-5},   # 底层慢
    {"params": model.lm_head.parameters(),    "lr": 1e-4},   # 头部快
])
```

#### 2. 训练轮数（防止过拟合）

微调数据量通常较小，轮数不宜多。

| 数据量 | 推荐 epochs |
|--------|------------|
| < 1000 条 | 3~5 |
| 1000~10000 条 | 2~3 |
| > 10000 条 | 1~2 |

#### 3. 学习率调度（warmup）

预训练模型权重已收敛，warmup 避免初期大幅震荡：

```python
TrainingArguments(
    warmup_ratio=0.05,        # 前 5% 步数线性升温
    lr_scheduler_type="cosine",  # cosine 衰减比 linear 更平滑
)
```

#### 4. LoRA 的 r 值（影响微调容量）

| 任务复杂度 | 推荐 r 值 |
|-----------|----------|
| 简单风格迁移 | 4~8 |
| 领域知识注入 | 16~32 |
| 复杂任务适配 | 64~128 |

### 需要避免的问题及对应参数

| 问题 | 原因 | 解决参数 |
|------|------|---------|
| 灾难性遗忘 | 学习率太大 | 降低 `learning_rate`，使用 LoRA |
| 过拟合 | 训练轮数太多 | 减少 `num_train_epochs`，增大 `weight_decay` |
| 训练不稳定/loss 震荡 | 梯度过大 | 降低 `max_grad_norm`（设 0.3~0.5） |
| 显存溢出 | batch 太大 | 减小 `per_device_train_batch_size`，增大 `gradient_accumulation_steps` |
| 模型输出退化 | 数据质量差或学习率过高 | 检查数据 + 降低 `learning_rate` |

### 完整微调参数示例（以 Qwen2.5 为例）

```python
from transformers import TrainingArguments
from peft import LoraConfig, get_peft_model, TaskType

# LoRA 配置
lora_config = LoraConfig(
    task_type=TaskType.CAUSAL_LM,
    r=16,                    # 中等复杂任务
    lora_alpha=32,
    lora_dropout=0.05,
    target_modules=["q_proj", "k_proj", "v_proj", "o_proj",
                    "gate_proj", "up_proj", "down_proj"],
)

# 训练参数
training_args = TrainingArguments(
    output_dir="./qwen_finetuned",
    num_train_epochs=3,                  # 小数据集用 3 轮
    per_device_train_batch_size=2,       # 显存不足时设小
    gradient_accumulation_steps=8,       # 等效 batch=16
    learning_rate=2e-4,                  # LoRA 微调推荐值
    warmup_ratio=0.05,                   # 前 5% 步预热
    lr_scheduler_type="cosine",
    weight_decay=0.01,
    max_grad_norm=0.3,                   # 微调时适当降低
    fp16=False,
    bf16=True,                           # Qwen 推荐 bfloat16
    logging_steps=10,
    save_strategy="epoch",
    evaluation_strategy="epoch",
    load_best_model_at_end=True,         # 自动加载最优 checkpoint
)
```

### 微调 vs 预训练参数对比

| 参数 | 预训练 | 微调 |
|------|--------|------|
| `learning_rate` | `1e-3 ~ 1e-4` | `1e-5 ~ 2e-4` |
| `num_train_epochs` | 数十~数百 | 1~5 |
| `batch_size` | 512~4096 | 4~32 |
| `warmup_ratio` | 0.01 | 0.03~0.1 |
| `weight_decay` | 0.1 | 0.01 |

## 八、参数调节速查表

| 目标 | 调节方向 |
|------|---------|
| 输出更有创意/多样 | 提高 `temperature`，提高 `top_p` |
| 输出更保守/精确 | 降低 `temperature`（0.1~0.3），关闭 `do_sample` |
| 减少重复内容 | 提高 `repetition_penalty`，设置 `no_repeat_ngram_size` |
| 显存不足 | 使用 4bit 量化 + LoRA，减小 `batch_size`，开启 `fp16` |
| 微调效果差 | 增大 `r`（LoRA），增加 `num_train_epochs`，调低 `learning_rate` |
| 训练不稳定 | 降低 `learning_rate`，增加 `warmup_steps`，降低 `max_grad_norm` |
| 过拟合 | 增大 `weight_decay`，增加 `dropout`，减少 `num_train_epochs` |
