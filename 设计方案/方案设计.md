# 边缘计算设备部署小模型方案设计

## 背景

在计算资源受限的边缘设备（如树莓派、Jetson Nano/Orin、手机 SoC、工业嵌入式板）上运行小型语言模型，需要在**性能、精度、速度、功耗**之间取得最优平衡。不同设备规格差异大，参数调节策略需因设备而异。

---

## 一、边缘设备分级与对应策略

| 设备等级 | 典型设备 | RAM | 推荐模型规模 | 核心策略 |
|---------|---------|-----|------------|---------|
| 高端边缘 | Jetson Orin、RK3588 | 8~16GB | 7B（量化） | 4bit 量化 + 流式推理 |
| 中端边缘 | Jetson Nano、RK3566 | 2~4GB | 1B~3B（量化） | 4bit 量化 + 裁剪 |
| 低端边缘 | 树莓派4、ESP32-S3 | 512MB~1GB | < 1B 或蒸馏模型 | INT8/INT4 + 极限裁剪 |

---

## 二、重点关注的参数（按优先级）

### 1. 量化参数（最优先，收益最大）

量化是边缘部署最重要的手段，可将显存/内存需求降低 50%~75%。

| 参数 | 说明 | 边缘设备推荐值 |
|------|------|-------------|
| `load_in_4bit` | INT4 量化 | 内存 ≤ 4GB 时首选 |
| `load_in_8bit` | INT8 量化 | 内存 4~8GB，精度要求较高时 |
| `bnb_4bit_compute_dtype` | 计算精度 | `torch.float16`（无 bfloat16 支持时）|
| `bnb_4bit_quant_type` | 量化算法 | `"nf4"`（精度优于 fp4）|
| `bnb_4bit_use_double_quant` | 双重量化 | `True`，额外节省约 0.4 bits/参数 |

```python
from transformers import AutoModelForCausalLM, BitsAndBytesConfig
import torch

# 针对边缘设备的量化配置
bnb_config = BitsAndBytesConfig(
    load_in_4bit=True,
    bnb_4bit_compute_dtype=torch.float16,
    bnb_4bit_use_double_quant=True,
    bnb_4bit_quant_type="nf4",
)

model = AutoModelForCausalLM.from_pretrained(
    "Qwen/Qwen2.5-1.5B-Instruct",  # 优先选小参数量模型
    quantization_config=bnb_config,
    device_map="auto",
    low_cpu_mem_usage=True,          # 降低加载时的峰值内存
)
```

### 2. 生成参数（直接影响推理速度和内存）

| 参数 | 边缘设备推荐值 | 原因 |
|------|-------------|------|
| `max_new_tokens` | **64~256** | 限制输出长度，避免长时间占用资源 |
| `do_sample` | **False**（贪婪）或 True | 贪婪解码速度最快，无需随机采样开销 |
| `num_beams` | **1** | beam search 内存占用是 beams 倍，边缘设备设 1 |
| `temperature` | **0.7**（采样时） | 按需设置，贪婪解码时此参数无效 |
| `top_k` | **20~50** | 减少候选计算量 |
| `use_cache` | **True** | 开启 KV cache，复用注意力计算，加速生成 |

```python
# 边缘设备推理参数配置
output = model.generate(
    **inputs,
    max_new_tokens=128,      # 限制输出长度
    do_sample=False,         # 贪婪解码，速度最快
    num_beams=1,             # 禁用 beam search
    use_cache=True,          # 必须开启 KV cache
    pad_token_id=tokenizer.eos_token_id,
)
```

---

### 3. 模型加载参数（影响内存占用）

| 参数 | 推荐设置 | 说明 |
|------|---------|------|
| `torch_dtype` | `torch.float16` | 半精度，内存减半，大多数边缘设备支持 |
| `device_map` | `"auto"` 或 `"cpu"` | 无 GPU 时强制 CPU |
| `low_cpu_mem_usage` | `True` | 逐层加载，降低加载峰值内存 |
| `offload_folder` | `"./offload"` | 内存不足时将部分层卸载到磁盘 |

```python
# 内存极度受限时（如 1~2GB RAM）
model = AutoModelForCausalLM.from_pretrained(
    "model_name",
    torch_dtype=torch.float16,
    device_map="auto",
    low_cpu_mem_usage=True,
    offload_folder="./offload",      # 超出内存的层写入磁盘
    offload_state_dict=True,
)
```

### 4. 针对不同设备规格的动态适配策略

根据设备实际内存自动选择参数，适配不同规格的边缘设备：

```python
import psutil
import torch

def get_device_config():
    """根据设备内存自动选择量化策略"""
    ram_gb = psutil.virtual_memory().total / (1024 ** 3)
    has_gpu = torch.cuda.is_available()
    gpu_mem_gb = torch.cuda.get_device_properties(0).total_memory / (1024**3) if has_gpu else 0

    if gpu_mem_gb >= 8 or ram_gb >= 8:
        # 高端边缘：8bit 量化，保留较好精度
        return {
            "quantization": BitsAndBytesConfig(load_in_8bit=True),
            "max_new_tokens": 256,
            "do_sample": True,
        }
    elif gpu_mem_gb >= 4 or ram_gb >= 4:
        # 中端边缘：4bit 量化
        return {
            "quantization": BitsAndBytesConfig(
                load_in_4bit=True,
                bnb_4bit_compute_dtype=torch.float16,
                bnb_4bit_use_double_quant=True,
                bnb_4bit_quant_type="nf4",
            ),
            "max_new_tokens": 128,
            "do_sample": False,
        }
    else:
        # 低端边缘：极限压缩，建议换用更小模型
        return {
            "quantization": BitsAndBytesConfig(
                load_in_4bit=True,
                bnb_4bit_use_double_quant=True,
                bnb_4bit_quant_type="nf4",
            ),
            "max_new_tokens": 64,
            "do_sample": False,
        }

config = get_device_config()
```

### 5. 微调参数（在边缘场景下定制模型）

如果需要在边缘设备的目标任务上微调（通常在服务器完成，再部署到边缘）：

| 参数 | 边缘场景推荐值 | 说明 |
|------|-------------|------|
| `r`（LoRA） | **4~8** | 小 r 值，微调后模型变化小，保持原有能力 |
| `lora_alpha` | `r * 2` | 保持缩放比例 |
| `target_modules` | 仅 `q_proj`, `v_proj` | 只改注意力，减少参数量 |
| `num_train_epochs` | **1~3** | 小数据集，避免过拟合 |
| `learning_rate` | **1e-4** | 小学习率，轻微适配 |
| `per_device_train_batch_size` | **1~2** | 微调时显存受限 |

```python
# 边缘场景的最小化 LoRA 配置
lora_config = LoraConfig(
    r=4,                              # 最小秩，参数量极小
    lora_alpha=8,
    target_modules=["q_proj", "v_proj"],  # 只改两个模块
    lora_dropout=0.05,
    bias="none",
    task_type=TaskType.CAUSAL_LM,
)
```

## 三、边缘部署推荐工具链

| 工具 | 用途 | 适用场景 |
|------|------|---------|
| `llama.cpp` | CPU 推理，极低内存占用 | 无 GPU 的边缘设备 |
| `ONNX Runtime` | 跨平台推理加速 | ARM、x86 边缘设备 |
| `MLC LLM` | 手机/嵌入式 GPU 推理 | 手机 SoC、Jetson |
| `BitsAndBytes` | 量化加载 | 有 CUDA 支持的设备 |
| `ctransformers` | 纯 CPU C++ 推理 | 无 GPU 场景 |

## 四、参数调节优先级总结

```
边缘设备部署调优顺序：

1. 选择合适的模型规模（1B~3B 优于 7B）
        ↓
2. 量化（4bit > 8bit > fp16）
        ↓
3. 限制生成长度（max_new_tokens）
        ↓
4. 关闭 beam search（num_beams=1）
        ↓
5. 贪婪解码（do_sample=False）
        ↓
6. 开启 KV cache（use_cache=True）
        ↓
7. 按设备规格动态适配参数
```




1. 设备分级 - 按内存把边缘设备分为高/中/低三档，对应不同模型规模和策略。

  2. 五类重点参数（按优先级）：
  - 量化参数 - 4bit/8bit，收益最大，内存直降 50~75%
  - 生成参数 - 限制 max_new_tokens、禁用 beam search、开启 KV cache
  - 模型加载参数 - low_cpu_mem_usage、offload_folder（内存极限时卸载层到磁盘）
  - 动态适配策略 - 根据设备实际内存自动选择参数的代码
  - 微调参数 - 边缘场景用最小 LoRA（r=4，只改两个模块）

  3. 推荐工具链 - llama.cpp、ONNX Runtime、MLC LLM 等按场景推荐。

  4. 调优顺序图 - 明确从选模型到参数调节的优先级流程。
